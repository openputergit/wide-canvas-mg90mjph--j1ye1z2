Technical Specification - MVP

Database Schema

users
├── id (PK)
├── email
├── phone
├── type (student/school_admin/super_admin)
├── name
├── city
├── academic_background
├── design_interests
└── created_at

schools
├── id (PK)
├── name
├── description
├── courses
├── location
├── contact_info
├── admin_id (FK -> users)
└── status

events
├── id (PK)
├── school_id (FK -> schools)
├── title
├── description
├── date
├── time
├── venue
├── city
└── capacity

event_registrations
├── id (PK)
├── event_id (FK -> events)
├── user_id (FK -> users)
└── status

content_posts
├── id (PK)
├── school_id (FK -> schools)
├── content_type
├── content
└── created_at

follows
├── id (PK)
├── user_id (FK -> users)
└── school_id (FK -> schools)

API Endpoints

  • POST /api/auth/register
  • POST /api/auth/verify
  • GET /api/schools
  • GET /api/schools/:id
  • POST /api/events
  • GET /api/events
  • POST /api/events/register
  • POST /api/posts
  • GET /api/feed
